## 📋 Требования

### 1. Разделение кода

**Сервер (отдельный пакет):**
```
chat-server/
├── server.js          # Node.js + Express + Socket.IO
├── config.json        # Порт, хост, пути к БД
├── database/
│   └── chat.db        # Общая БД
├── uploads/           # Файлы
└── install-server.bat # Скрипт установки
```

**Клиент (Electron приложение):**
```
chat-client/
├── electron/
│   ├── main.js        # Только окно + подключение
│   └── preload.js
├── frontend/          # React
│   └── src/
│       ├── App.js
│       └── components/
│           ├── ServerConnect.js  # 🆕 Экран подключения
│           └── Login.js
└── package.json
```

---

### 2. Серверная часть

**Технологии:**
- Node.js 16+
- Express.js
- Socket.IO
- SQLite (потом опционально MongoDB)

**Конфигурация (config.json):**
```json
{
  "server": {
    "host": "0.0.0.0",
    "port": 3001
  },
  "database": {
    "path": "./database/chat.db"
  },
  "uploads": {
    "path": "./uploads",
    "maxFileSize": 52428800
  },
  "security": {
    "corsOrigins": ["http://*"],
    "jwtSecret": "your-secret-key"
  }
}
```

**Функционал:**
- REST API для авторизации, сообщений, файлов
- WebSocket для real-time уведомлений
- JWT аутентификация
- CORS для сетевого доступа
- Загрузка файлов
- Админ-панель (будущая версия)

## 📦 Установщики

### Сервер (NSIS):
- `chat-server-setup.exe`
- Устанавливает Node.js зависимости
- Создает службу для автозапуска (опционально)
- Открывает порт в брандмауэре
- Создает конфиг

### Клиент (NSIS):
- `chat-client-setup.exe`
- Устанавливает Electron приложение
- Без серверных компонентов
- Легче текущего (~50MB vs ~136MB)

## 🚀 Варианты развертывания сервера

### Вариант A: На компьютер в офисе
```bash
# На компьютере который всегда включен (например, у админа)
1. Скачать chat-server-setup.exe
2. Установить
3. Запустить "Chat Server"
4. Запомнить IP адрес (например, 192.168.1.50)
5. Открыть порт 3001 в брандмауэре Windows
```

**Плюсы:**
- Бесплатно
- Быстро

**Минусы:**
- Зависит от компьютера
- Динамический IP (может меняться)

---

### Вариант B: Арендовать VPS (рекомендую)
**Провайдеры:**
- Timeweb Cloud (~500 руб/мес)
- Reg.ru (~600 руб/мес)
- Selectel (~700 руб/мес)
- AWS/Azure (дороже)

**Плюсы:**
- Статический IP
- Работает 24/7
- Не зависит от офисных ПК
- Доступ из любой точки

**Минусы:**
- Стоимость (~500-700 руб/мес)

---

### Вариант C: Гибридный (временный)
1. Сервер на ПК пока нет VPS
2. Потом миграция на VPS
3. Клиенты переподключаются к новому URL

## 📌 Заметки

### Текущие проблемы (для решения в будущем):
1. Electron приложение не загружает frontend (фиолетовый экран)
2. `electron-updater` требует `node_modules` в сборке
3. Пути к ресурсам в production

### Решено отложить:
- Автообновления клиента (вернется после разделения)
- HTTPS (когда будет VPS)
- MongoDB (SQLite достаточно для 100 пользователей)
