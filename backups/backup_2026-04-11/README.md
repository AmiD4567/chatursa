# 💬 Чат-приложение с SQLite (RocketChat-подобный)

Приложение для обмена сообщениями и файлами в реальном времени с поддержкой личных и групповых чатов, с сохранением данных в SQLite базе.

## Возможности

- ✅ **Личные чаты** - общение один на один
- ✅ **Групповые чаты** - создание чатов с несколькими участниками
- ✅ **Общий чат** - доступен всем пользователям
- ✅ **Передача файлов** - изображения, документы, видео, аудио
- ✅ **Превью изображений** - отображение картинок в чате
- ✅ **Статусы пользователей** - онлайн/оффлайн индикаторы
- ✅ **Счетчик непрочитанных** - уведомление о новых сообщениях
- ✅ **Push-уведомления** - браузерные уведомления о новых сообщениях
- ✅ **Идентификация пользователей** - запоминание по email и паролю
- ✅ **SQLite база данных** - сохранение всех данных после перезапуска
- ✅ **Адаптивный дизайн** - работает на мобильных устройствах

## База данных

База данных SQLite включает таблицы:

| Таблица | Описание |
|---------|----------|
| `users` | Пользователи (ID, username, avatar, IP, host, статус) |
| `chats` | Чаты (ID, тип, название, создатель) |
| `chat_participants` | Участники чатов |
| `messages` | Сообщения (текст, файлы, timestamp) |
| `unread_messages` | Непрочитанные сообщения |

## Запуск

### 1. Установка зависимостей backend

```bash
cd chat-app/backend
npm install
```

> **Примечание:** Для Windows может потребоваться установка `windows-build-tools`:
> ```bash
> npm install --global --production windows-build-tools
> ```

### 2. Запуск backend

```bash
npm start
```

Сервер запустится на `http://localhost:3001`
База данных создастся автоматически: `backend/chat.db`

### 3. Установка зависимостей frontend (в новом окне терминала)

```bash
cd chat-app/frontend
npm install
```

### 4. Запуск frontend

```bash
npm start
```

Приложение откроется на `http://localhost:3000`

## Использование

1. Откройте браузер и перейдите на `http://localhost:3000`
2. Введите ваше имя и нажмите "Войти"
3. В боковой панели отображаются все доступные чаты
4. Для создания нового чата нажмите ✏️ в разделе "Чаты"
5. Выберите тип чата (личный или групповой) и участников
6. Отправляйте сообщения и файлы в выбранный чат

### Повторный вход

При повторном входе с того же браузера/устройства:
- Вы автоматически войдете под своим именем
- Вся история сообщений сохранится
- Статус пользователя обновится на "онлайн"

## Технологии

- **Backend:** Node.js, Express, Socket.IO, Multer, better-sqlite3, bcryptjs
- **Frontend:** React, Socket.IO Client
- **База данных:** SQLite3
- **Стили:** CSS3

## API

### Сокет события

**Клиент → Сервер:**
- `join` - присоединение пользователя (с userId для повторного входа)
- `create_chat` - создание нового чата
- `join_chat` - присоединение к чату
- `send_message` - отправка сообщения
- `get_users` - получение списка пользователей
- `typing` - статус набора текста

**Сервер → Клиент:**
- `user_joined_success` - успешное присоединение (с userId для сохранения)
- `chat_history` - история сообщений чата
- `new_message` - новое сообщение
- `chat_created` - создан новый чат
- `chat_updated` - обновлен чат (новое сообщение/счетчик)
- `users_list` - список пользователей
- `user_status_changed` - изменение статуса пользователя
- `user_typing` - пользователь печатает

### REST API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/upload` | Загрузка файла |
| GET | `/api/user/:userId` | Информация о пользователе |
| GET | `/uploads/:filename` | Доступ к загруженным файлам |

## Хранение данных

- **Пользователи:** Сохраняются с уникальным ID, привязанным к IP и User-Agent
- **Сообщения:** Все сообщения сохраняются в БД с timestamp
- **Файлы:** Сохраняются в папке `uploads/`, в БД хранится метаданные
- **Сессия:** userId сохраняется в localStorage браузера

## Очистка данных

Для полного сброса данных удалите файл базы данных:
```bash
rm backend/chat.db
# или вручную удалите файл через проводник
```
