# 📋 Rocket.Chat Архитектура — Анализ для нашего чата

## Как работает Rocket.Chat

### 1. Серверная часть (Server)
```
┌──────────────────────────────────────────┐
│         Rocket.Chat Server               │
│  ┌────────────────────────────────────┐  │
│  │  MongoDB (база данных)             │  │
│  ├────────────────────────────────────┤  │
│  │  Node.js сервер                    │  │
│  │  - REST API                        │  │
│  │  - WebSocket (Real-time)           │  │
│  │  - File storage                    │  │
│  └────────────────────────────────────┘  │
│                                          │
│  Доступен по: https://chat.company.com   │
│  или: http://192.168.1.100:3000          │
└──────────────────────────────────────────┘
```

**Установка сервера:**
- Docker: `docker run rocket.chat`
- Snap: `sudo snap install rocketchat-server`
- Manual: Download + Node.js + MongoDB
- Cloud: Rocket.Chat Cloud (SaaS)

**Настройка сервера:**
1. Указать host/IP где будет доступен
2. Настроить порт (обычно 3000)
3. Настроить MongoDB
4. Создать админа
5. Настроить SSL (опционально)

---

### 2. Клиентская часть (Client Apps)
```
┌─────────────────────────────────────────────┐
│  Rocket.Chat Client (Desktop/Mobile)        │
│  ┌───────────────────────────────────────┐  │
│  │  Экран подключения                    │  │
│  │  ┌─────────────────────────────────┐  │  │
│  │  │ Server URL:                     │  │  │
│  │  │ [https://chat.company.com]      │  │  │
│  │  │            [Connect]            │  │  │
│  │  └─────────────────────────────────┘  │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│  После подключения:                         │
│  ┌───────────────────────────────────────┐  │
│  │  Login / Password                     │  │
│  │  [username] [password] [Sign In]      │  │
│  └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

**Клиенты:**
- Desktop (Windows/Mac/Linux)
- Mobile (iOS/Android)
- Web (browser)

**Клиент НЕ содержит:**
- ❌ Базы данных
- ❌ Backend сервера
- ❌ Файлового хранилища

**Клиент содержит:**
- ✅ UI для подключения к серверу
- ✅ Авторизация
- ✅ Общение через WebSocket/REST
- ✅ Кэширование сообщений

# ЗАДАЧА: Рефакторинг чата в клиент-серверную архитектуру (как Rocket.Chat)

## ТЕКУЩАЯ АРХИТЕКТУРА
- Electron приложение содержит frontend + backend + SQLite
- Всё работает внутри одного .exe
- Проблема: сложно обновлять, масштабировать, использовать в сети

## ЦЕЛЕВАЯ АРХИТЕКТУРА (как Rocket.Chat)

### 1. Сервер (separate installation)
- Node.js + Express + Socket.IO
- SQLite (или MongoDB для масштабирования)
- REST API + WebSocket
- Установка:
  - .exe установщик для Windows
  - Docker образ
  - npm install + node server.js
- Конфигурация:
  - PORT (по умолчанию 3001)
  - HOST (0.0.0.0 для сети или localhost)
  - DB_PATH (путь к базе данных)
  - UPLOADS_PATH (путь к файлам)
- Админка для управления пользователями

### 2. Клиент (Electron приложение)
- Только frontend (React)
- Экран подключения к серверу:
  - Ввод Server URL (http://server:port)
  - Кнопка "Подключиться"
  - Сохранение последних серверов
- После подключения:
  - Логин/пароль
  - Основная функциональность чата
- Без базы данных (или кэш для офлайн-режима)
- Без backend кода

## ЭТАПЫ РЕФАКТОРИНГА

### Этап 1: Разделение кода
1.1 Выделить backend в отдельный репозиторий/пакет
1.2 Выделить frontend (Electron client) в отдельный пакет
1.3 Убрать SQLite из Electron приложения
1.4 Настроить API calls вместо локальных вызовов

### Этап 2: Серверная часть
2.1 Создать установщик сервера (NSIS)
2.2 Добавить конфигурационный файл (config.json)
2.3 Добавить сервис/демон для автозапуска
2.4 Добавить админ-панель для управления

### Этап 3: Клиентская часть
3.1 Добавить экран выбора сервера
3.2 Сохранение списка серверов в localStorage
3.3 Подключение к выбранному серверу
3.4 Обработка отключений/ошибок

### Этап 4: Сетевая функциональность
4.1 CORS настройки на сервере
4.2 HTTPS поддержка
4.3 Аутентификация через JWT tokens
4.4 Reconnection логика

## ПРИОРИТЕТЫ

🔴 Критично (MVP):
- Разделение backend/frontend
- Экран подключения к серверу
- Сервер работает независимо

🟡 Важно:
- Установщик сервера
- Установщик клиента
- Сохранение нескольких серверов

🟢 Желательно:
- Docker образ сервера
- HTTPS настройка
- Офлайн кэш

## ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Сервер:
- Node.js 16+

- [ ] Работает в локальной сети
- [ ] Работает через интернет (с HTTPS)

## 🎯 Рекомендации для нашего случая

### Вариант A: Полное разделение (как Rocket.Chat)
**Для кого:** Корпоративные клиенты, несколько офисов

1. **Сервер** устанавливается на отдельный сервер/компьютер
2. **Клиенты** устанавливаются пользователям
3. Подключение по сети (LAN/Internet)

**Плюсы:**
- Масштабируемость
- Централизованное управление
- Общая история чатов
- Легкие обновления клиента

**Минусы:**
- Нужен выделенный сервер
- Сложнее настройка сети/HTTPS

---

### Вариант B: Гибридный (рекомендую для начала)
**Для кого:** Малый бизнес, один офис

1. **.exe приложение** содержит и сервер и клиент
2. Сервер запускается автоматически
3. Клиент подключается к localhost
4. **Опция:** Подключение к удаленному серверу

**Плюсы:**
- Простая установка (один .exe)
- Работает из коробки
- Опция для роста (удаленный сервер)

**Минусы:**
- Сервер на каждом компьютере
- Нет общей истории

---
