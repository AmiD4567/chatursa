# Красная точка на иконке при непрочитанных сообщениях

## Что реализовано

Теперь при наличии непрочитанных сообщений в чате, на иконке приложения в **трее** и **панели задач** отображается красная точка с числом непрочитанных сообщений.

## Как это работает

### Frontend (App.js)
- Добавлен `useEffect` который суммирует все `unreadCount` из всех чатов
- При изменении суммы, отправляет общее количество в основной процесс Electron через `window.electronAPI.setUnreadCount(totalUnread)`
- При открытии чата, `unreadCount` этого чата сбрасывается в 0, что автоматически обновляет общую сумму

### Основной процесс Electron (main.js)

#### Функция `createBadgeIcon(count)`
- Создает иконку с красной точкой в правом верхнем углу
- Использует offscreen BrowserWindow с Canvas API для рисования
- Красная точка имеет:
  - Градиент от светло-красного к темно-красному
  - Белую обводку для контраста
  - Тень для объемности
  - Число непрочитанных сообщений внутри (до 99, далее "99+")
- Возвращает Promise с объектом `NativeImage`

#### Функция `updateUnreadBadge()`
- Асинхронная функция которая вызывает `createBadgeIcon()`
- Устанавливает иконку как:
  - **Overlay icon** на окне (отображается в панели задач Windows)
  - **Иконка трея** (в системном трее)
- При счетчике 0 - возвращает стандартную иконку без бейджа

### Preload (preload.js)
- Предоставляет мост `window.electronAPI.setUnreadCount(count)` для отправки счетчика

## Визуальный результат

- **Красная точка** в правом верхнем углу иконки
- **Число** внутри точки показывает количество непрочитанных сообщений
- Если сообщений > 99, отображается "99+"
- Точка анимирована с градиентом и тенью для лучшего визуального эффекта

## Технические детали

### Рисование через Offscreen Window
Поскольку в Node.js нет прямого доступа к Canvas API, используется подход с созданием временного невидимого BrowserWindow:
1. Создается окно размером с иконку (32x32 или больше)
2. Загружается HTML с canvas элементом
3. На canvas рисуется базовая иконка + красная точка с числом
4. Результат отправляется обратно в основной процесс через IPC
5. Окно закрывается

### Обработка параллельных обновлений
Добавлен `badgeUpdatePromise` для предотвращения одновременных обновлений бейджа. Если несколько обновлений приходят быстро, они ожидают завершения предыдущего.

## Тестирование

1. Запустите приложение: `npm run electron:dev`
2. Войдите в систему
3. Отправьте себе сообщение из другого аккаунта (не открывая чат)
4. Проверьте что:
   - На иконке в трее появилась красная точка с числом
   - На иконке в панели задач появилась красная точка с числом
5. Откройте чат - бейдж должен исчезнуть
6. Отправьте несколько сообщений в разные чаты - число должно суммироваться
