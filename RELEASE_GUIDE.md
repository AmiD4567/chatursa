# 🚀 Выпуск новой версии (Release)

## Подготовка

1. **Увеличьте версию** в `package.json`:
   ```json
   "version": "1.0.10"  // было "1.0.9"
   ```

2. **Обновите CHANGELOG.md** (опционально, но рекомендуется):
   ```markdown
   ## [1.0.10] - 2026-04-08
   ### Добавлено
   - Система автообновления
   - Видео-фон на странице входа
   
   ### Исправлено
   - Эмодзи теперь отображаются корректно
   - Первое сообщение доставляется сразу
   ```

## Получение GitHub токена (один раз)

1. Зайдите на GitHub → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
2. Нажмите **Generate new token (classic)**
3. Название: `ChatAppRelease`
4. Галочка: ✅ `public_repo` (или `repo` для приватных)
5. Нажмите **Generate token** и **скопируйте** его (начинается с `ghp_...`)

⚠️ **Важно:** Токен показывается только один раз! Сохраните его в надёжном месте.

## Публикация релиза

### Windows (PowerShell):
```powershell
$env:GH_TOKEN="ghp_ваш_токен_здесь"
npm run release
```

### Windows (CMD):
```cmd
set GH_TOKEN=ghp_ваш_токен_здесь
npm run release
```

### Linux/Mac:
```bash
export GH_TOKEN=ghp_ваш_токен_здесь
npm run release
```

## Что происходит при публикации:

1. ✅ Собирается frontend (`npm run build`)
2. ✅ Создаётся MSI-установщик
3. ✅ Создаётся новый Release на GitHub
4. ✅ Загружается `Chat App-x.x.x.msi`
5. ✅ Создаётся `latest.yml` (для проверки версий)

## Как пользователи получают обновление:

1. Приложение автоматически проверяет GitHub при запуске
2. Если есть новая версия — показывает уведомление
3. Пользователь нажимает "Скачать обновление"
4. После загрузки — "Перезапустить и установить"
5. MSI устанавливается автоматически

## Проверка релиза:

После публикации проверьте:
- https://github.com/pantyuhov/chat-app/releases
- Там должен появиться новый тег и MSI-файл

## Безопасность:

- 🔒 **Никогда** не коммитьте `.env` с реальным токеном
- 🔒 Токен уже добавлен в `.gitignore`
- 🔒 Используйте `.env.example` как шаблон
- 🔒 Храните токен в менеджере паролей
